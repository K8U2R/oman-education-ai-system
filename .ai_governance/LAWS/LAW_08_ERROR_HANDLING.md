# القانون 08: بروتوكول الفشل الآمن (Fail-Safe Protocol)

## التعريف (Definition)

يجب التعامل مع جميع الأخطاء بشكل صريح. يُحظر ترك الاستثناءات (Exceptions) دون معالجة، ويمنع استخدام `console.log` للأخطاء في الإنتاج.

## السبب التقني (The Rationale)

1. **الأمان:** رسائل الخطأ الافتراضية قد تسرب معلومات عن البنية التحتية.
2. **المراقبة:** السجلات الموحدة (Structured Logs) تسهل تتبع المشاكل.

## القائمة البيضاء (المسموح ✅)

- استخدام فئات أخطاء مخصصة: `throw new UserNotFoundException()`.
- استخدام `logger.error()` مع السياق (Metadata).

## القائمة السوداء (المحظور ❌)

```typescript
// محظور: ابتلاع الخطأ
try { ... } catch (e) { console.log(e) }
```

## آلية التنفيذ (Enforcement)

- قاعدة ESLint: `no-console`.
